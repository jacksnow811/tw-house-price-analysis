## ✅ 已解決的核心問題（關鍵原理）

之前一直卡在「按搜尋沒反應」，原因是網站的搜尋按鈕 **不是送 AJAX**，而是一段
JS handler，流程如下：

```js
var webdriver = window.navigator.webdriver;
if (webdriver) return;          // ← 1. 偵測自動化瀏覽器，是的話直接結束
... // 2. 驗證必填欄位（含「交易標的」ptype 至少勾一個）
localStorage.setItem("form-data", JSON.stringify(dataObj));  // 3. 條件存 localStorage
location.href = "list.jsp";     // 4. 導向 list.jsp，由它讀 localStorage 呈現結果
```

**之前所有點擊都無效的真正原因**：Playwright 會把 `navigator.webdriver` 設為 true，
handler 第一行就 `return` 了，根本沒往下跑。

### 兩個關鍵修正
1. **偽裝 webdriver**：`context.add_init_script` 注入
   `Object.defineProperty(navigator,'webdriver',{get:()=>undefined})`
2. **勾選「交易標的」(ptype)**：必填欄位，`input[name='ptype']` 全部勾選
   （房地=1,2／土地=3／建物=4／車位=5）

修正後：JS 原生 `button[go_type="list"].click()` → handler 正常執行 →
index.jsp frame 導向 `list.jsp` → 結果出現在 `#price_table`。

## 明細資料（每列「明細」按鈕）
點 `#price_table` 每列的 `button.detail`（明細）會：
- 觸發 AJAX `GET /SERVICE/QueryPrice/detail/{id}/{token}`，回傳 JSON
- 並彈出 modal `#detailModalCenter`（**必須關掉才能點下一筆**，否則遮罩擋住點擊）

JSON 欄位對應（已加進 CSV 每筆後面）：
| CSV 欄位 | JSON 路徑 |
|----------|-----------|
| 主建物坪數 | `buildlist.主建物` |
| 陽台坪數 | `buildlist.陽台` |
| 車位類別 | `r[].r2` |
| 車位價格 | `r[].r3` |
| 車位面積 | `r[].r4` |
| 所在樓層 | `r[].r6` |
（多車位時以「；」串接；以 /detail/ 回應直接解析，不依賴 modal 內容）

**關 modal 的關鍵**：要先等 modal 完全 `show`（動畫結束）再強制關閉，
否則在開啟動畫途中關閉會被 Bootstrap transition 覆寫，導致 modal 卡住擋下一筆。
見 `close_detail_modal()`。
