## Context

本專案已有抓取（`scraper.py`）與分析（`analysis.py`，核心為「實坪制坪價 ＝（總價 − 車位價格）÷ 主建物坪數」）。`housing-analysis` skill 負責「這個建案值不值得買」的質性評價；但**沒有任何工具回答「該出多少價」**。

使用者已明確選擇：(1) 資料來源**只讀 `data/` 的 CSV**；(2) 行情知識**每次即時上網查**；(3) 輸出為**完整報告**（三段式報價＋話術＋風險＋行情比較表）。本設計據此規劃 `price-negotiation` skill 的架構。

## Goals / Non-Goals

**Goals:**

- 以社區實價登錄 comps（實坪制坪價）為定價錨點，對單一**大樓**物件產出對買方最有利、且**站得住腳（可佐證）**的三段式報價。
- 把「樓層／景觀／車位／主建物坪數／總坪數（公設比）／屋齡／地段」等因子，量化成相對 comps 的加減價調整。
- 每次執行即時上網補當下區域行情與因子幅度，與 comps 互相校準。
- 輸出可直接拿去跟房仲／屋主談的話術與風險清單。

**Non-Goals:**

- 不研究透天、土地、店面、套房、廠辦——僅限電梯／住宅大樓。
- 不新增 Python 程式或套件、不改 `scraper.py`／`analysis.py`。
- 不做賣方估價、不做投資報酬試算、不提供法律／貸款意見。
- 不在 skill 內建固定行情數字當唯一依據（行情走即時查詢；參考檔只放「方法論＋起點區間」）。

## Decisions

**D1：定價錨點用「實坪制坪價」，且每車均價以單一社區自算。**
沿用 `analysis.py` 公式，但 skill 只針對**目標社區那一個 CSV** 計算，並在該社區內自算「平均每車價格」（取車位獨立販售 comps 的均值）來拆非獨立販售車位。理由：`analysis.py` 會合併 `data/` 全部社區、跨社區平均每車價格，談單一社區時會被別區污染。談價要的是「同社區可比性」，故 scope 到單一社區最準。替代方案（直接讀 `analysis/analysis_實坪制坪價.csv` 再篩 `來源檔`）較省事但車位均價被跨社區污染，故不採。

**D2：行情走即時 WebSearch/WebFetch，參考檔只放方法論與起點區間。**
依使用者選擇，每次執行查當地區域行情與因子加減價幅度（如「每高一層約 +1~3%」「機械車位約平面 5~7 折」「屋齡每年約 −2%」）。參考檔的數字僅是**起點 orientation，必須以即時查詢與 comps 校準**，避免行情過期。替代方案（內建固定表）被使用者否決。

**D3：「對買方最有利」＝三段式報價，全部錨在 comps 分位數＋買方市場議價空間。**
先用 comps 算出「與目標物件條件相稱」的實坪制坪價區間（中位數與分位數），加上即時查到的議價空間（2025–2026 七都約 10–12% 且擴大），導出：
- **起始出價**：積極壓低、但仍可用 comps 下緣／物件缺點佐證（避免被仲介一句「亂出」打發）。
- **目標成交價**：買方理想落點，約相稱 comps 的偏低分位。
- **心理天花板**：可走人價（walk-away），不超過相稱 comps 合理上緣。
三個價位都以「實坪制坪價 × 目標主建物坪數 ＋ 車位合理價」回推總價，讓買方知道每個數字怎麼來。

**D4：嚴格限定大樓，先辨識標的型態再談。**
以 CSV 的 `型態`／`主要用途`／`交易標的`，加上使用者描述的物件，判斷是否為電梯／住宅大樓。非大樓（透天、土地、店面、套房、廠辦）→ 明確說明不在範圍、不給價，並建議改用適當方法。

**D5：工具邊界。**
讀 `data/` 與算統計用 Bash（`uv run python` ＋ pandas）或直接 Read（社區資料通常數十至數百筆，量小可直接讀）；即時行情用 WebSearch／WebFetch。`allowed-tools` 因此含 `Read, Glob, Bash, WebSearch, WebFetch`。

## Risks / Trade-offs

- **comps 過少或社區近期無量** → 拉長到 1–2 年、放寬可比條件，並在報告標明信心下降；真的無資料則請使用者先用 `scraper.py` 抓或改指定檔案，不臆測。
- **即時行情噪音／新聞農場** → 多來源互相印證、剔除明顯特殊交易，區分「查到的事實」與「我的推論」並附連結。
- **出價過於積極傷誠信／談判破局** → 起始出價必須能用 comps 或物件缺點佐證；報告同時給「目標」與「天花板」讓買方自己拿捏，並提醒過低出價的風險。
- **實價登錄特殊交易污染**（親友、含裝潢、毛胚、急售、車位灌價） → 篩選時剔除離群與特殊註記，並在報告說明剔除了哪些。
- **與 `analysis.py` 公式漂移** → 參考檔註明公式以 `analysis.py` 為準，未來改公式需同步。
