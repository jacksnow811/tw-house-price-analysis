# remark/ 社區景觀備註（對內／對外）

`price-negotiation` skill 用這裡的檔案，把每個社區的成交分成兩類：

- **對外**：朝大馬路／公園／河、永久棟距，視野開闊 → 通常**貴很多**。
- **對內**：朝中庭／天井／鄰棟 → 通常較便宜。

skill 會用每筆成交的「**地段位置或門牌**」去比對你寫的規則，把 comps 分成兩組，
再用實坪制坪價中位數差算出這個社區自己的「**對外溢價**」，讓出價更準。

## 怎麼放

- **一個社區一個檔**：`remark/<社區名>.md`（檔名只要**含**社區名即可，跟 `data/` 的 CSV 一樣用關鍵字比對）。
- `README` 開頭、`_` 開頭（範本）、`.` 開頭的檔會被略過，不會被當成某個社區的備註。

## 規則格式（一行一條）

```
側別 | 地址關鍵字 | 座向／說明
```

- **側別**：`對外` 或 `對內`。
- **地址關鍵字**：會出現在實價登錄「地段位置或門牌」裡的字串，通常是 **路／街／巷／弄／棟**
  （例：`建德路`、`12巷`、`A棟`）。
  - ⚠ 實價登錄的「**號**」**有時**會被去識別化成區間（如 `1~30號`），這時請盡量用**不會被遮蔽**的
    路/巷/弄/棟層級；同一門牌區間內若還要再分對內外，skill 分不出來、會標「**未知**」。
  - 若門牌**完整保留號與戶別**（如 `…段１６號七樓之５`，數字常為**全形**），可搭配下面的 `&` 同時鎖定。
- **`&` 多重關鍵字（AND）**：地址關鍵字可用 `&` 串接多段，代表門牌需**同時包含**全部才算命中
  （例：`段１６號 & 之５` ＝門牌同時含『段１６號』與『之５』）。常用於「號」與「之X 戶別」中間夾著
  樓層、無法用單一連續字串鎖定時。提醒：`２號` 是 `１２號` 的子字串，故請在號前加「段」等字錨定。
- **座向／說明**：給人看的理由（朝公園、面中庭…）；skill 不解析，但有助你維護。
- `#` 開頭、或整行沒有 `|` 的行 = 註解／自由筆記，skill 會略過。
- 一筆成交命中多條規則時，**較精確者優先**（命中子字串**長度和**較大者；故 `A & B` 勝過只寫 `A`，
  可用「棟別預設＋指定戶覆蓋」分層）；若兩條不同側別精確度相同 → 標「**未知**」並提示衝突。
- 也吃 markdown 表格寫法（`| 對外 | 建德路 | 面公園 |`），表頭與 `---` 分隔列會自動略過。

## 範例

見 `remark/_範本.md`，複製後改名成 `remark/<你的社區名>.md` 再填即可。

## 跑起來看

```bash
uv run python .claude/skills/price-negotiation/scripts/comps_stats.py "<社區名>" --years 2
```

輸出會多一段「【景觀：對外 vs 對內】」：兩組中位數、對外溢價、各樓層帶兩側對照，
以及「未命中規則的門牌」清單（可據此回頭補規則）。沒有備註檔就會略過並提示可建立。
